# 🐷 นิพนธุ์ฟาร์ม (Nipon Farm) - Project Documentation

เอกสารฉบับนี้จัดทำขึ้นเพื่อให้นักพัฒนา (Developer) สามารถทำความเข้าใจภาพรวมของโปรเจกต์ โครงสร้างระบบ และสถานะปัจจุบันของแอปพลิเคชัน "นิพนธุ์ฟาร์ม" ได้อย่างรวดเร็ว

## 1. ภาพรวมโปรเจกต์ (Project Overview)
แอปพลิเคชันสำหรับบริหารจัดการฟาร์มสุกรแบบครบวงจร (Web Application) ออกแบบมาเพื่อใช้งานผ่านสมาร์ทโฟนและแท็บเล็ตเป็นหลัก (Mobile-responsive) โดยมุ่งเน้นที่ความรวดเร็วในการบันทึกข้อมูลหน้าฟาร์ม การจัดการวงจรชีวิตแม่พันธุ์ (Sow Management) และการทำบัญชีตั๋วชั่ง/การขายสุกร (Sales Management)

## 2. เทคโนโลยีหลักที่ใช้ (Tech Stack)
* **Frontend:** React 18, TypeScript, Vite
* **Styling:** Tailwind CSS (เน้นความเรียบง่าย ใช้พื้นหลังสี Slate อ่อน และการ์ดสีขาว ตามความต้องการของผู้ใช้งาน)
* **Icons:** `lucide-react`
* **Routing/State:** React state เบื้องต้น และ Hooks แบบ Custom
* **Backend & Database:** Firebase เต็มรูปแบบ (บริการทั้งหมดของ Google)
  * **Authentication:** Google Sign-in Login
  * **Database:** Cloud Firestore (NoSQL)
  * **Storage:** Firebase Storage (แพลนกำลังจำนำมาปรับใช้กับระบบสแกนบิลด้วย OCR)

## 3. ระบบและฟีเจอร์ปัจจุบัน (Current Features)

### 3.1 ระบบจัดการแม่หมู (Sow Management)
จัดการวงจรชีวิตแม่หมูและติดตามสถานะในแต่ละวัน
* **รายชื่อและสถานะ (Sow List):**
  * แสดงข้อมูลเป็นรูปแบบการ์ด (Card View) แยกกันชัดเจน
  * ฟิลเตอร์กรองตามสถานะต่างๆ เช่น แม่หมูสาว, รอผสม, อุ้มท้อง, คลอด/เลี้ยงลูก, หย่านม, หมูปลด
  * เรียงลำดับ (Sorting) ได้หลายแบบ เช่น เรียงตามรหัส, รอบการผลิต (Parity), วันที่เข้าเล้า ฯลฯ
* **การเพิ่มแม่หมู (Add Sow):**
  * บันทึกรหัสแม่หมู, สายพันธุ์, วันที่เข้าเล้า, วันเกิด
  * *(Ongoing)* กำลังปรับปรุงให้เมื่อเพิ่มแม่หมูใหม่ จะเซ็ตสถานะเริ่มต้นเป็น **"แม่หมูสาว"** ทันที และมีสีสถานะเฉพาะตัว
* **รายละเอียดวงจรและการผสม (Sow Details):**
  * บันทึกรอบการผลิต (Parity) และกำหนดการต่างๆ
  * *(Ongoing)* กำลังปรับระบบให้สามารถ **"กดยืนยันการผสม" ได้หลายครั้งในรอบเดียวกัน** (เช่น รอบเช้า, รอบเย็น หรือวันเว้นวัน) แทนที่แบบเดิมที่บันทึกได้แค่ครั้งเดียว เพื่อความแม่นยำในการเก็บข้อมูลประวัติน้ำเชื้อที่ใช้
* **หน้าปฏิทินแจ้งเตือน (Calendar View):**
  * แสดงจุดสีแจ้งเตือนกำหนดการต่างๆ ของแม่หมูในฟาร์ม (เช่น เตรียมคลอด, หย่านม, เช็กสัดซ้ำ, ตรวจท้อง) วันต่อวัน

### 3.2 ระบบการขายและการเงิน (Sales & Financial Management)
ระบบจัดการการออกตั๋วชั่งเพื่อขายสุกรหน้าฟาร์ม (Sale Form Components)
* **รูปแบบการขาย:** รองรับการตั้งค่าประเภทการขายคือ 1. ขายเหมา 2. ขายชั่งกิโล 3. หมูปลด
* **ฟอร์มการขาย (แบ่งเป็น 3 Tab ป้องกันความสับสน):**
  1. **ข้อมูลทั่วไป:** ข้อมูลผู้ซื้อ (มีระบบ Autocomplete จำชื่อผู้ซื้อเดิม), ผู้ขาย, ทะเบียนรถ, วันที่
  2. **ระบบชั่งน้ำหนัก (Weighing):** 
     * สามารถเพิ่มรายการชั่งน้ำหนักในแต่ละบิลได้หลายรอบ
     * หน้าจอใหญ่ พิมพ์ง่าย (Numpad-friendly)
     * มีช่องใส่น้ำหนักรวม (Gross Weight), หักกรง (Tare Weight) และคำนวณน้ำหนักสุทธิ (Net Weight) ออกมาให้อัตโนมัติในแต่ละรอบชั่ง
  3. **สรุปยอด (Payment / Summary):** 
     * คำนวณยอดเงินรวมจากค่าเฉลี่ยและน้ำหนักสุทธิทั้งหมด
     * สามารถใส่หมายเหตุหรือ "ยอดหักลดหย่อน/หักอื่นๆ" ออกมาเป็นยอดชำระสุทธิ (Net Total)
     * รองรับฟังก์ชันการกด "พิมพ์ (Print)" เพื่อพริ้นท์สลิปได้ทันที

### 3.3 แดชบอร์ดและสถิติ (Dashboard & Analytics)
* แสดงสรุปตัวเลขแม่หมูทั้งหมด และแยกตามรายสถานะ
* วัดประสิทธิภาพการผลิตผ่าน **"สถิติลูกเกิดรอดเฉลี่ย (Live born)"** และ **"สถิติลูกหย่านมเฉลี่ย (Weaned)"** ต่อครอก

### 3.4 ระบบผู้ใช้งานและโปรไฟล์ (User Profile)
* สมัครสมาชิกและเข้าสู่ระบบด้วย Google Account (มีความปลอดภัยสูง ไม่ต้องจำรหัสผ่าน)
* Modal จัดการโปรไฟล์ผู้ใช้ อัปโหลดและเปลี่ยนรูปโปรไฟล์ได้
* (ระบบออกแบบมารองรับการแยกข้อมูลของแต่ละฟาร์ม/ผู้ใช้ ผ่าน `user.uid` ของ Firebase)

## 4. สิ่งที่กำลังจะพัฒนาเพิ่มเติม (Upcoming Roadmap)

1. **ปรับปรุงระบบบันทึกการผสมพันธุ์ (Mating System):** 
   * รองรับ Multiple Mating Records ต่อ 1 รอบการติดสัด (เพิ่มการผสมหลายครั้ง เช่น เช้า, เย็น)
   * รองรับการเก็บรหัสหรือชนิดของน้ำเชื้อแยกกันในแต่ละครั้ง
2. **ระบบบันทึกรายจ่ายและ OCR (Expense OCR Tracking):** 
   * ใช้งานร่วมกับ AI (Gemini)
   * เมื่อผู้ใช้อัปโหลดรูปสลิปหรือบิลค่าอาหาร/ยา รููปจะถูกอัปโหลดขึ้น Firebase Storage
   * ส่ง URL ไปให้ AI ถอดรหัสข้อมูลออกมาเป็น Text เพื่อบันทึกรายจ่ายเข้าบัญชีฟาร์มอัตโนมัติ (อยู่ในขั้นตอนรอเจ้าของโปรเจกต์ตั้งค่า Billing ใน Firebase)

## 5. การติดตั้งและการรันในเครื่องส่วนตัวสำหรับนักพัฒนา (Developer Setup)

1. Clone repository
2. ติดตั้ง Dependencies:
   ```bash
   npm install
   ```
3. ตรวจสอบไฟล์ `.env` สำหรับ Firebase Configuration. (ในระบบใช้ไฟล์ `firebase-applet-config.json` สำหรับการตั้งค่าเชื่อมต่อ Firebase)
4. เริ่มการทำงานเซอร์เวอร์โหมดพัฒนา:
   ```bash
   npm run dev
   ```

*หมายเหตุ: โปรเจกต์นี้ไม่ได้ใช้การแสดงผล Background แบบ App-Theme (#F2F2F7/ #F2EBE1) ตาม Requirement ล่าสุด สีพื้นหลังได้ถูก Revert กลับมาเป็น Standard Tailwind UI (bg-slate-50, bg-white) เป็นส่วนใหญ่*
