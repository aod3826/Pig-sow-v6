=== เอกสารความต้องการระบบ (System Requirement Specification) ===
ระบบ: โมดูลการขายหมูขุนหน้าฟาร์ม (Smart Pig Sales Module)
สถานะ: สมบูรณ์และผ่านการทดสอบหน้างานจริง

1. ภาพรวมระบบ (System Overview)
แอปพลิเคชันสำหรับให้พนักงานใช้โทรศัพท์มือถือหรือแท็บเล็ตบันทึกการชั่งน้ำหนักหมูขุนขึ้นรถแบบ Real-time โดยเน้นความเร็วในการป้อนข้อมูล ป้องกันข้อมูลสูญหายจากอุบัติเหตุ และออกใบเสร็จ PDF/Email ได้ทันที

2. โครงสร้างข้อมูล (Data Model)
ข้อมูลการขาย (Sale Record) ประกอบด้วย:
- วันที่ขาย (Date), เลขที่อ้างอิง (ID)
- ชื่อผู้ซื้อ (Buyer Name), อีเมลผู้ซื้อ (Buyer Email - Optional)
- ทะเบียนรถ (Vehicle Registration)
- รูปแบบการขาย (Sale Type) เช่น ขายเหมา
- สถานะการชำระเงิน (Payment Status): "เงินสด" (PAID) หรือ "ค้างชำระ" (UNPAID)
- จำนวนหมูทั้งหมด (Total Pigs)
- ราคาต่อกิโลกรัม (Price per Kg), รายการหักค่าใช้จ่าย (Deductions)
- ลายเซ็นผู้ซื้อ (Signature - Base64 Image)

ข้อมูลการชั่งน้ำหนักแต่ละตัว/รอบ (Weighing Record) ประกอบด้วย:
- ลำดับ (Index)
- น้ำหนักรวม (Gross Weight)
- น้ำหนักกรง (Tare Weight) 
- น้ำหนักสุทธิ (Net Weight) -> คำนวณอัตโนมัติ (Gross - Tare) ไม่ติดลบ

3. ระบบป้องกันข้อมูลสูญหาย (Fail-safe & Anti-Data Loss) *สำคัญมาก*
- Auto-Save Draft: ทุกครั้งที่มีการป้อนน้ำหนักหมู, พิมพ์ชื่อ หรือกรอกตัวเลข ระบบต้องบันทึกสถานะทั้งหมดลงใน Local Storage (เช่น คีย์ `pig_sale_draft`) แบบ Real-time
- Accidental Exit Prevention: หากผู้ใช้มีตะกร้าชั่งหมูค้างไว้ แล้วเผลอปัดหน้าจอทิ้ง หรือกดปุ่ม Back ของเบราว์เซอร์ ต้องมี Pop-up ของระบบ (beforeunload) ขวางไว้ไม่ให้ออก
- Crash Recovery: ถ้าระบบล่ม บราวเซอร์พัง พอกลับเข้าแอปใหม่ ระบบต้องเช็ค Local Storage ทันที ถ้าพบข้อมูลค้างอยู่ ต้องขึ้น Alert ถามว่า "พบข้อมูลชั่งค้างอยู่ ต้องการทำต่อหรือไม่?" ถ้ากดทำต่อ ให้โหลดข้อมูลที่ชั่งไว้ทั้งหมดแถมค้างกลับมาที่เดิมเป๊ะ

4. ระบบป้อนข้อมูลความเร็วสูง (Speed Data Entry) สำหรับหน้างาน
- การป้อนและขึ้นตัวใหม่: เมื่อพิมพ์ตัวเลขในช่อง "น้ำหนักรวม" หรือ "หักกรง" เสร็จ ผู้ใช้สามารถกดปุ่ม "Enter" (หรือปุ่ม Next ในคีย์บอร์ดมือถือ) ระบบจะต้องทำการสร้างแถวใหม่ (หรือกล่องใหม่) ให้อัตโนมัติ โดยที่มือไม่ต้องละจากคีย์บอร์ดเพื่อไปทัชปุ่ม "+ เพิ่ม"
- Auto-Carryover: เมื่อสร้างแถวใหม่/กล่องใหม่ ระบบจะต้องดึงค่า "น้ำหนักกรง" ของตัวล่าสุด มาใส่รอไว้ให้ในตัวถัดไปทันที (ผู้ใช้จะได้พิมพ์แค่น้ำหนักรวมอย่างเดียวเพราะกรงเดิม)
- View Model: หน้าป้อนรายการชั่งต้องมี 2 มุมมอง (สามารถสลับได้):
   - Table View: สรุปเป็นตาราง เหมาะสำหรับ Tablet/PC
   - Card View: รูปแบบกล่องขนาดใหญ่ เหมาะสำหรับกดด้วยนิ้วโป้งบน Smartphone

5. การคำนวณทางคณิตศาสตร์ (Calculations)
- น้ำหนักสุทธิรวม (Total Net Weight) = ผลรวมของ Net Weight ทุกรายการชั่ง
- น้ำหนักเฉลี่ย (Average Weight) = Total Net Weight / จำนวนหมูทั้งหมด
- ยอดรวม (Gross Total) = Total Net Weight * ราคาต่อหน่วย (Price per Kg)
- ยอดสุทธิ (Net Total) = Gross Total - รายการหัก (Deductions)

6. ระบบ PDF ใบชั่งน้ำหนัก (PDF Receipt Generation)
- ขนาดและหน้าตา: จัดรูปแบบให้อยู่ในหน้า A4 (หรือเทียบเท่า ความกว้าง 794px สำหรับ 96dpi) รูปแบบเรียบง่าย ไม่ใช้ Layout ซับซ้อนแบบคอลัมน์เกินจำเป็น
- ข้อจำกัดเชิงเทคนิค: การ Generate PDF (แนะนำใช้ html2canvas + jsPDF) **ห้าม**ใช้ CSS ฟังก์ชันสีขั้นสูง เช่น oklch() หรือตัวแปรสีแปลกๆ ต้องเขียนสไตล์การให้สีแบบระบุค่า Hex Code ตรงๆ เท่านั้น (เช่น color: '#000000', backgroundColor: '#ffffff') ป้องกันปัญหา Canvas แตก หรือเส้นกรอบทับซ้อนกัน

7. ระบบการแจ้งเตือนและการส่งออก (Export & Notify)
- รูดลายเซ็น: ก่อนบันทึกการขาย ระบบต้องบังคับให้มี Digital Signature ของผู้รับหมู
- กดปุ่มอีเมล: ถ้าระบุอีเมลผู้ซื้อไว้ จะมีปุ่มส่งอีเมล โดยระบบใช้เมธอด mailto: ร่างข้อความอัตโนมัติแจ้งสรุป (จำนวนหมู, น้ำหนักรวม, น้ำหนักเฉลี่ย, ราคา, ยอดสุทธิ, สถานะชำระ)
- ปุ่มดาวน์โหลด: มีปุ่มจำเพาะสำหรับสร้างและ Download PDF ลงเครื่องโดยตรง
